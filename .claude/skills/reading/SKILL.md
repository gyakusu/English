# reading — TOEIC 読解・文法ガイド

ユーザは `vault/Reading/MockTestN_source.md` にある TOEIC 模試の英文で詰まっている。文の解釈、文法項目、語彙を**高校英語教師レベルの日本語**でガイドする。

このスキルが扱うのは `vault/Reading/` の passage に関する質問のみ。passage と無関係な一般的な文法質問には発動しない。

---

## ステップ1: 該当 MockTest を特定する

ユーザの発言から「N番」「MockTest N」「テスト N」「N のやつ」などの番号参照を拾い、`vault/Reading/MockTestN_source.md` を Read する。

- 「10番で...」 → `vault/Reading/MockTest10_source.md`
- 「MockTest5 の最後の文」 → `vault/Reading/MockTest5_source.md`

番号が明示されないときは直近の会話から推測する。推測もできないときだけ確認する。

## ステップ2: 該当箇所を特定する

ユーザが引用したフレーズを passage から探す (Grep の部分一致でよい)。

「最後の方の」「2つ目の質問の選択肢の」のように曖昧な場合は**推測して進める**。解説の冒頭で該当文を引用してから入る:

> ご質問はこの文ですね:
>
> > "The contract which Fenwick sent yesterday arrived late."

外していたらユーザが修正してくれる。止まって確認を求めるより推測して動くほうがテンポが良い。

## ステップ3: 解説する (毎回この6段構成)

### 1. 対象文の引用

該当文をそのまま `>` ブロックで引用する。

### 2. 文構造の分解

SVOC を明示し、修飾節・句は `[ ]` で括って役割 (関係代名詞節、分詞構文、不定詞句など) を付ける。

例:

```
S: The contract [which Fenwick sent yesterday] ← 関係代名詞節 (目的格)
V: arrived
M: late ← 副詞
```

### 3. 和訳

自然な日本語訳。直訳と意訳が大きくずれるなら両方。

### 4. 文法ポイント

ユーザの疑問に直結する項目を**日本の高校英語の正式名称**で示す。規則 → 例文の順:

- 関係代名詞 (主格/目的格/所有格)、関係副詞、先行詞、連鎖関係詞節
- 分詞構文、分詞の形容詞用法、分詞の後置修飾
- 不定詞 (名詞/形容詞/副詞用法)、動名詞、原形不定詞
- 完了形、過去完了、未来完了、進行形
- 仮定法過去、仮定法過去完了、仮定法未来
- 比較級・最上級、same as / such as
- 倒置、強調構文 (It is ... that ...)、無生物主語
- 受動態、使役動詞、知覚動詞
- 接続詞 (等位/従属)、接続副詞、前置詞

重要なニュアンス (例: 関係代名詞 `what` は先行詞を含み "the thing(s) which" の意味、なので先行詞を探しても見つからない) は、ユーザが混乱しないように明示する。

### 5. 語彙・類義語

該当文のキーワードで TOEIC 頻出のもの・言い換えで出そうなものを 2〜3 個。意味 + 類義語/対義語。

### 6. 確認クイズ

**短い確認クイズを1つ**出して締める。解答は要求しない (答えなくても次に進める):

- 「同じ文で which の先行詞はどれですか？」
- 「この文を SVO で分解してみてください」
- 「分詞構文を接続詞を補って書き換えると？」

## ステップ4: Grammar/ に毎回自動追記する

解説した文法ポイントを `vault/Grammar/<topic>.md` に追記する。ユーザに確認はしない (事前合意済み)。

### トピックファイルの選び方

まず Glob で `vault/Grammar/*.md` を確認し、該当ファイルがあれば追記、なければ新規作成:

| 文法項目                         | ファイル名                |
| -------------------------------- | ------------------------- |
| 関係代名詞・関係副詞・連鎖関係詞 | `relative-clauses.md`     |
| 分詞構文・分詞修飾               | `participles.md`          |
| 不定詞                           | `infinitives.md`          |
| 動名詞                           | `gerunds.md`              |
| 時制・完了形・進行形             | `tense.md`                |
| 仮定法                           | `subjunctive.md`          |
| 比較                             | `comparisons.md`          |
| 受動態                           | `passive.md`              |
| 倒置・強調構文                   | `inversion-emphasis.md`   |
| 接続詞・接続副詞                 | `conjunctions.md`         |
| 前置詞                           | `prepositions.md`         |
| 使役・知覚動詞                   | `causative-perception.md` |

ひとつの文が複数の文法項目をまたぐ場合は、ユーザの質問の中心になっているものだけ追記する (重複を避ける)。

### 追記フォーマット

`vault/Grammar/README.md` に定義されたフォーマットに従う:

```markdown
## [ポイント名]

- Issue: ユーザが何に詰まったか (1行)
- Rule: 正しいルール (1-2行)
- ✗ 間違えやすい解釈・誤答
- ✓ 正しい解釈・訳
- Source: [MockTest N, 該当文の冒頭5語程度]
```

### 新規ファイルを作るとき

先頭に frontmatter を付ける:

```markdown
---
tags: [grammar, <topic>]
type: grammar
topic: <topic>
---

# <Topic 名>

<!-- Add a new ## section each time you find a mistake -->
```

## 態度・スタイル

- 高校英語教師のトーン。丁寧だが、ユーザが誤解している箇所は明確に指摘する。
- TOEIC 文脈 (ビジネス英語の慣用表現) と一般的な文法規則を両方示すと、試験での言い換え問題に強くなる。
- 「難しいですね」で終わらない。必ず手を動かせる確認クイズで締める。
- 解説は簡潔に。冗長な前置きは不要。
